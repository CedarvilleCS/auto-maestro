## Overview

The MCP (Model Context Protocol) Server is the controlled execution layer between AutoMaestro agents and the isolated SEED Lab sandbox environment.

It enforces execution mediation and ensures that all commands generated by AI agents are:

- Routed through a structured API
- Executed only inside isolated containers
- Captured and normalized before being returned
- Never executed directly on uncontrolled infrastructure

The MCP server is a core architectural safeguard supporting AutoMaestro’s reliability model.

---

## Responsibilities

The MCP server provides a REST API that allows agents and developers to:

- Discover running sandbox containers and their IP addresses
- Perform connectivity checks (`/ping`)
- Execute commands inside sandbox containers (`/exec`)
- Retrieve container metadata (`/containers`)
- Return structured JSON responses to requesting agents

All execution is strictly confined to the sandbox environment.

---

## Architectural Role

The MCP server supports AutoMaestro’s second reliability pillar:

**Sandbox-Constrained Execution**

It ensures:

- Execution Isolation  
- Structured Output Normalization  
- Controlled Command Routing  
- Reproducible Testing Conditions  

This design prevents unintended consequences from AI-generated commands.

## Command Notes

- /containers returns a list of valid sandbox container names and IPs
- /ping -> performs network connectivity tests between containers
- /exec -> executes arbitrary commands inside a specified sandbox container
- mcp logs -> run `docker logs mcp-server`to view mcp server logs

## Security & Scope
The MCP server:
- Only executes commands inside the sandbox environment
- Does not expose direct host-level execution
- Should not be exposed publicly without additional authentication and hardening
This component is intended for controlled lab and research environments
