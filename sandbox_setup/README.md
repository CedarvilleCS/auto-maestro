# Sandbox Setup (SEED Lab Environment)

This directory contains the Docker-based SEED Lab environment used by AutoMaestro to simulate realistic cyber infrastructure.

The sandbox provides a controlled, reproducible network where AI-generated commands can be executed safely and evaluated without interacting with live systems.

---

## Architectural Role

The sandbox implements AutoMaestro’s second reliability pillar:

### Sandbox-Constrained Execution

All executable commands generated by agents:

1. Are routed through the backend
2. Passed to the MCP server
3. Executed inside isolated Docker containers
4. Captured and normalized before being returned

No agent interacts directly with host infrastructure.

This ensures:

- Execution isolation  
- Reproducibility  
- Structured output reintegration  
- Safe experimentation  

---

## Overview

The sandbox environment:

- Simulates attacker and target networks
- Provides configurable vulnerable services
- Enables realistic tool testing (Nmap, Hydra, Metasploit, etc.)
- Supports network-level communication between containers
- Is fully containerized via Docker Compose

Because the Docker configuration is modular, the SEED Lab topology can be modified or replaced without affecting agent orchestration logic.

## Quick Start

From the project root (recommended):

```bash
./scripts/up.sh
```

To stop:

```bash
./scripts/down.sh
```

### Starting Only the Sandbox

From inside `sandbox_setup/`:

```bash
docker-compose -f docker-compose.seed.yml up -d --build
```

---

## Hosts

The `Hosts/` directory contains Dockerfiles for simulated SEED lab hosts.

Host containers may represent:

- Attacker machines
- Defender/target systems
- Vulnerable web or database servers
- Utility nodes for network simulation

Hosts can be customized by modifying Dockerfiles or adding configuration scripts.

This modularity allows experimentation with:

- Different vulnerability scenarios
- Alternate network topologies
- Tool-specific test cases

---
## Verify Setup

To confirm containers are running correctly:

```bash
./test_containers.sh
```

You can also verify:

```bash
docker ps
```

---

## Security & Scope

The sandbox:

- Is designed exclusively for controlled lab environments
- Should not be exposed publicly
- Should not be connected to production infrastructure
- Assumes container-level isolation is enforced by Docker

AutoMaestro does not execute commands outside this sandbox environment.
